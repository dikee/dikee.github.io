Keeping track of Python processes

  • Mon 02 November 2015
  • ·
  • Python
  • ·
  • python, supervisor

I'm a big fan of managing long running Python processes with supervisor. At the very least, supervisor will automatically restart any failed Python process (like your gunicorn server).

On Ubuntu, simply sudo apt-get install supervisor.

Assuming we're deploying a Flask app named weather_app, we can create a file in /etc/supervisor/conf.d/weather_app.conf with the content below.

[weather_app]
command=/path/to/python /path/to/gunicorn weather_app:app
directory=/path/to/weather_app 
autorestart=true

Thats it. Its a super basic setup but it'll get an app running.

The final step is to use the supervisor command line tool, supervisorctl, to update and start weather_app.

supervisorctl reread
supervisorctl update
supervisorctl start weather_app

Type ps aux | grep weather_app in the terminal and you should see the running process.
