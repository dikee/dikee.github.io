In the last post, I walked through how to persist the stock index in a file.

This post goes over how to automatically run that script each day at midnight using a cronjob.

Instructions on how to do this can be found here.

A summary is:

Type crontab -e to open the cron config file.

Type in a cron command in the file.

The format of the command follows this format:

## source: https://help.ubuntu.com/community/CronHowto
minute (0-59), hour (0-23, 0 = midnight), day (1-31), month (1-12), weekday (0-6, 0 = Sunday), command

In our case, if the python file is saved at /var/projects/stock_index.py, you would add the command below to the cron config file.

00 00 * * * python /var/projects/stock_index.py

Save the file and you're all set. Every day at midnight, the script will run and update the archive.txt file shown in the previous post.

To ensure we can find the output file, designate a full file path for the archive file from the last post.

Simply change

with open('archive.txt', 'a') as the_file:

to

with open('/path/to/desired/directory/archive.txt', 'a') as the_file:

In my case, I used /var/projects/archive.txt in order to have the archive file in the same destination as the script.
